This guide will walk you through setting up a complete development environment for **7POINTS.NET (Pisukide)**, the shared apartment management platform built with Laravel, React, and Odoo.

<Note>
  This installation guide is based on the comprehensive Basque README documentation and has been adapted for English-speaking developers.
</Note>

### Hardware Requirements

- **CPU**: 64-bit processor with virtualization support (Intel Core i5 or equivalent)
- **RAM**: Minimum 8 GB (16 GB recommended for Docker)
- **Storage**: At least 50 GB free space (SSD recommended)
- **OS**: Windows 10/11 (64-bit), Linux (Ubuntu, Debian), or macOS

<Warning>
  For Windows users: **WSL 2** (Windows Subsystem for Linux) must be enabled for Docker Desktop.
</Warning>

### Step 1: Install PHP 8.2

#### Windows (PowerShell as Administrator)

Check if PHP is installed and verify the version:

```powershell
php -v
```

If PHP 8.2+ is not installed, run:

<CodeGroup>
```powershell Download PHP 8.2
# Download PHP 8.2
Invoke-WebRequest -Uri "https://windows.php.net/downloads/releases/archives/php-8.2-Win32-vs16-x64.zip" -OutFile "$env:USERPROFILE\Downloads\php82.zip"
```

```powershell Extract PHP
# Create PHP directory and extract
New-Item -ItemType Directory -Path "C:\php82" -Force
Expand-Archive -Path "$env:USERPROFILE\Downloads\php82.zip" -DestinationPath "C:\php82"
```

```powershell Add to PATH
# Add PHP to system PATH
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\php82", [EnvironmentVariableTarget]::Machine)
```

```powershell Verify Installation
# Verify PHP installation
php -v
# Expected output: PHP 8.2.x (cli) (built: ...)
```
</CodeGroup>

#### Configure PHP for Laravel

```powershell
# Copy and open php.ini
Copy-Item "C:\php82\php.ini-development" "C:\php82\php.ini"
notepad C:\php82\php.ini
```

In `php.ini`, **remove the semicolon (`;`)** from the beginning of these lines:

```ini
extension=curl
extension=mbstring
extension=bcmath
extension=xml
extension=mysqli
extension=openssl
extension=pdo_mysql
extension=fileinfo
```

Save and close the file.

#### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install -y php8.2 php8.2-cli php8.2-fpm php8.2-mysql php8.2-xml php8.2-mbstring php8.2-curl php8.2-zip php8.2-bcmath
php -v
```

#### macOS

```bash
brew install php@8.2
php -v
```

#### Run Database Migrations

```bash

```

This creates tables including:
- `users` - User accounts
- `pisua` - Apartments  
- `gastos` - Expenses
- `zereginak` - Tasks
- `ordainketas` - Payments
- `piso_user` - Apartment-user relationships

<CodeGroup>
```php Piso Model
// From: app/Models/Piso.php
class Piso extends Model
{
    protected $table = 'pisua';

    protected $fillable = [
        'izena',
        'kodigoa',
        'odoo_id',
        'synced',
        'sync_error',
        'user_id',
    ];

    public function inquilinos()
    {
        return $this->belongsToMany(User::class, 'piso_user', 'piso_id', 'user_id')
            ->withPivot('mota')
            ->withTimestamps();
    }

    public function gastos()
    {
        return $this->hasMany(Gasto::class, 'IdPiso', 'id');
    }
}
```

### Step 10: Start Docker Containers (Odoo + PostgreSQL)

Navigate back to the project root:

```bash
cd ..  # Back to project root
```

Start the Docker services:

```bash
docker compose up -d
```

This starts containers defined in `docker-compose.yaml`:

<CodeGroup>
```yaml Docker Compose Configuration
# From: docker-compose.yaml
version: '3.8'

services:
  # Laravel + PHP
  laravel_app:
    image: php:8.2-fpm-bullseye
    build:
      context: ./laravel
      dockerfile: Dockerfile
    container_name: laravel_app
    working_dir: /var/www/html
    volumes:
      - ./laravel:/var/www/html
    ports:
      - "8080:8080"
    command: bash -c "php artisan serve --host=0.0.0.0 --port=8080"
    networks:
      - app-network

  # Odoo ERP
  odoo:
    image: odoo:17.0
    depends_on:
      - postgres
    ports:
      - "8069:8069"
    volumes:
      - ./odoo/addons:/mnt/extra-addons
      - odoo_data:/var/lib/odoo
    environment:
      - HOST=postgres
      - USER=odoo
      - PASSWORD=myodoo
    networks:
      - app-network
      
  # PostgreSQL Database
  postgres:
    image: postgres:15
    ports:
      - 5432:5432
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=postgres
      - POSTGRES_PASSWORD=myodoo
      - POSTGRES_USER=odoo
    networks:
      - app-network

volumes:
  postgres_data:
  odoo_data:

networks:
  app-network:
    driver: bridge
```

```bash Start Docker Services
# Start all services
docker compose up -d

# Check running containers
docker compose ps

# View logs
docker compose logs -f odoo

# Stop services
docker compose down
```
</CodeGroup>

Verify services are running:

```bash
docker compose ps
```

You should see:
- `laravel_app` on port 8080
- `odoo` on port 8069  
- `postgres` on port 5432

<Warning>
  **First-time Odoo setup:** Navigate to `http://localhost:8069` and create the initial database with:
  - **Database name**: `postgres`
  - **Email**: `admin@example.com`
  - **Password**: `admin`
</Warning>

### Step 11: Run the Development Server

You have two options for running the development environment:

#### Option A: Using Docker (Recommended)

The Laravel app is already running in Docker on port 8080.

Start the Vite development server for hot reloading:

```bash
cd laravel
npm run dev
```

#### Option B: Run Laravel Locally

```bash
cd laravel

# Start Laravel development server
php artisan serve --host=0.0.0.0 --port=8080

# In another terminal, start Vite
npm run dev
```

#### Option C: Use Composer Scripts (Concurrently)

The project includes a convenient composer script:

```bash
composer run dev
```

This runs (from composer.json):
```json
"scripts": {
  "dev": [
    "Composer\\Config::disableProcessTimeout",
    "npx concurrently -c \"#93c5fd,#c4b5fd,#fdba74\" \"php artisan serve\" \"php artisan queue:listen --tries=1\" \"npm run dev\" --names='server,queue,vite'"
  ]
}
```

This starts:
- Laravel development server
- Queue worker (for Odoo sync jobs)
- Vite dev server with HMR

### Queue Workers

The application uses queues for Odoo synchronization:

```bash
# Run queue worker
php artisan queue:work

# Or use queue:listen for development (auto-reloads)
php artisan queue:listen --tries=1
```

Sync jobs include:
- `SyncUserToOdoo` - Sync users to Odoo partners
- `SyncPisoToOdoo` - Sync apartments to Odoo
- `SyncGastoToOdoo` - Sync expenses to Odoo
- `SyncZereginakToOdoo` - Sync tasks to Odoo

## Troubleshooting

### Common Issues

<Accordion title="Port 8080 already in use">
  **Solution 1:** Stop the conflicting service
  
  ```bash
  # Find process using port 8080
  lsof -i :8080  # macOS/Linux
  netstat -ano | findstr :8080  # Windows
  
  # Kill the process
  kill -9 <PID>
  ```
  
  **Solution 2:** Change the port
  
  ```bash
  php artisan serve --port=8000
  ```
</Accordion>

<Accordion title="Docker containers won't start">
  **Check Docker is running:**
  ```bash
  docker info
  ```
  
  **Check logs:**
  ```bash
  docker compose logs
  ```
  
  **Reset Docker:**
  ```bash
  docker compose down -v
  docker compose up -d
  ```
</Accordion>

<Accordion title="Composer install fails">
  **Clear Composer cache:**
  ```bash
  composer clear-cache
  composer install
  ```
  
  **Update dependencies:**
  ```bash
  composer update
  ```
</Accordion>

<Accordion title="npm install fails">
  **Clear npm cache:**
  ```bash
  npm cache clean --force
  rm -rf node_modules package-lock.json
  npm install
  ```
</Accordion>

<Accordion title="Odoo connection errors">
  **Verify Odoo is running:**
  ```bash
  docker compose ps
  docker compose logs odoo
  ```
  
  **Check .env configuration:**
  ```bash
  ODOO_URL=http://localhost:8069
  ODOO_DB=postgres
  ODOO_USERNAME=admin
  ODOO_PASSWORD=admin
  ```
  
  **Test Odoo API manually:**
  ```bash
  curl http://localhost:8069/web/database/list
  ```
</Accordion>

<Accordion title="Migration errors">
  **Reset database:**
  ```bash
  php artisan migrate:fresh
  ```
  
  **Rollback and re-run:**
  ```bash
  php artisan migrate:rollback
  php artisan migrate
  ```
</Accordion>
