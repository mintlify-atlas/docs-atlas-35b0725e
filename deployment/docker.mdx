## Overview

7POINTS.NET uses Docker to containerize the application stack, making deployment consistent across different environments. The application consists of three main services:

- **Laravel App**: PHP 8.2-based web application (port 8080)
- **Odoo**: ERP system for expense and task management (port 8069)
- **PostgreSQL**: Database server (port 5432)

## Docker Compose Configuration

The project uses `docker-compose.yaml` to orchestrate all services. Here's the complete configuration:

```yaml docker-compose.yaml
version: '3.8'

services:
  # Laravel + PHP
  laravel_app:
    image: php:8.2-fpm-bullseye
    build:
      context: ./laravel
      dockerfile: Dockerfile
    container_name: laravel_app
    working_dir: /var/www/html
    volumes:
      - ./laravel:/var/www/html
    ports:
      - "8080:8080"
    command: bash -c "php artisan serve --host=0.0.0.0 --port=8080"
    networks:
      - app-network

  # Odoo Service
  odoo:
    image: odoo:17.0
    depends_on:
      - postgres
    ports:
      - "8069:8069"
    volumes:
      - ./odoo/addons:/mnt/extra-addons
      - odoo_data:/var/lib/odoo
    environment:
      - HOST=postgres
      - USER=odoo
      - PASSWORD=myodoo
    networks:
      - app-network

  # PostgreSQL Database
  postgres:
    image: postgres:15
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=postgres
      - POSTGRES_PASSWORD=myodoo
      - POSTGRES_USER=odoo
    networks:
      - app-network

volumes:
  postgres_data:
  odoo_data:

networks:
  app-network:
    driver: bridge
```

## Services Breakdown

### Laravel Application

- **Base Image**: `php:8.2-fpm-bullseye`
- **Port Mapping**: 8080:8080
- **Volume Mount**: `./laravel` → `/var/www/html`
- **Command**: Runs Laravel's built-in development server
- **Network**: Connected to `app-network`

The Laravel Dockerfile installs necessary PHP extensions:

```dockerfile Dockerfile
FROM php:8.2-fpm-bullseye

# Install system dependencies for Laravel
RUN apt-get update && apt-get install -y \
    libpng-dev \
    libjpeg-dev \
    libfreetype6-dev \
    zip \
    git \
    && docker-php-ext-configure gd --with-freetype --with-jpeg \
    && docker-php-ext-install gd \
    && docker-php-ext-install pdo pdo_mysql

# Install Composer
RUN curl -sS https://getcomposer.org/installer | php -- \
    --install-dir=/usr/local/bin --filename=composer
```

### Odoo ERP System

- **Base Image**: `odoo:17.0`
- **Port Mapping**: 8069:8069
- **Volume Mounts**:
  - `./odoo/addons` → `/mnt/extra-addons` (custom addons)
  - `odoo_data` → `/var/lib/odoo` (session and file data)
- **Dependencies**: Requires PostgreSQL to start first
- **Network**: Connected to `app-network`

### PostgreSQL Database

- **Base Image**: `postgres:15`
- **Port Mapping**: 5432:5432
- **Volume Mount**: `postgres_data` → `/var/lib/postgresql/data`
- **Network**: Connected to `app-network`

## Starting Services

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone <repository-url>
    cd 7points-net
    ```
  </Step>

  <Step title="Start all services">
    ```bash
    docker-compose up -d
    ```
    
    The `-d` flag runs containers in detached mode (background).
  </Step>

  <Step title="Verify services are running">
    ```bash
    docker-compose ps
    ```
    
    You should see three services running:
    - laravel_app (port 8080)
    - odoo (port 8069)
    - postgres (port 5432)
  </Step>

  <Step title="Access the applications">
    - **Laravel**: http://localhost:8080
    - **Odoo**: http://localhost:8069
  </Step>
</Steps>

## Managing Services

### Stopping Services

```bash
# Stop all services
docker-compose down

# Stop and remove volumes (WARNING: deletes database data)
docker-compose down -v
```

### Restarting Services

```bash
# Restart all services
docker-compose restart

# Restart a specific service
docker-compose restart laravel_app
```

### Viewing Logs

```bash
# View all service logs
docker-compose logs

# View logs for specific service
docker-compose logs laravel_app
docker-compose logs odoo
docker-compose logs postgres

# Follow logs in real-time
docker-compose logs -f

# View last 100 lines
docker-compose logs --tail=100
```

### Executing Commands in Containers

```bash
# Access Laravel container
docker-compose exec laravel_app bash

# Run Artisan commands
docker-compose exec laravel_app php artisan migrate
docker-compose exec laravel_app php artisan db:seed

# Access PostgreSQL
docker-compose exec postgres psql -U odoo -d postgres
```

## Volumes and Data Persistence

The application uses Docker volumes to persist data:

- **postgres_data**: Stores PostgreSQL database files
- **odoo_data**: Stores Odoo session data and uploaded files
- **./laravel**: Bound mount for Laravel application code
- **./odoo/addons**: Bound mount for custom Odoo modules

<Warning>
  When running `docker-compose down -v`, all data in volumes will be deleted. Always backup your database before removing volumes.
</Warning>

## Network Configuration

All services are connected via the `app-network` bridge network. This allows:

- Services to communicate using container names as hostnames
- Laravel can connect to PostgreSQL using hostname `postgres`
- Odoo can connect to PostgreSQL using hostname `postgres`
- Isolation from other Docker networks

## Production Considerations

### Security

<Warning>
  The default configuration uses weak passwords. **Always change these in production**:
  - PostgreSQL password: `myodoo`
  - Odoo admin credentials
</Warning>

### Performance Optimization

1. **Use production PHP image**: Replace `php:8.2-fpm-bullseye` with optimized production image
2. **Enable OPcache**: Configure PHP OPcache for better performance
3. **Resource limits**: Set memory and CPU limits in docker-compose

```yaml
laravel_app:
  deploy:
    resources:
      limits:
        cpus: '1'
        memory: 512M
```

### Reverse Proxy

In production, use Nginx or Traefik as a reverse proxy:

```yaml
nginx:
  image: nginx:alpine
  ports:
    - "80:80"
    - "443:443"
  volumes:
    - ./nginx.conf:/etc/nginx/nginx.conf
  depends_on:
    - laravel_app
```

### Environment-Specific Configuration

Create separate docker-compose files:

```bash
# Development
docker-compose -f docker-compose.yml up

# Production
docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d
```

## Troubleshooting

### Port Already in Use

If ports 8080, 8069, or 5432 are already in use:

```yaml
ports:
  - "8081:8080"  # Change host port
```

### Container Fails to Start

```bash
# Check container logs
docker-compose logs laravel_app

# Rebuild containers
docker-compose build --no-cache
docker-compose up -d
```

### Database Connection Issues

Ensure PostgreSQL is ready before Laravel starts:

```yaml
laravel_app:
  depends_on:
    postgres:
      condition: service_healthy
```
