## Overview

The 7POINTS.NET application requires proper environment configuration for both Laravel and Odoo services. This guide covers all necessary environment variables and configuration steps.

## Laravel Environment Configuration

### Creating the .env File

Laravel uses a `.env` file in the `laravel/` directory for environment-specific configuration.

<Steps>
  <Step title="Copy example file">
    ```bash
    cd laravel
    cp .env.example .env
    ```
  </Step>

  <Step title="Generate application key">
    ```bash
    php artisan key:generate
    ```
    
    Or within Docker:
    ```bash
    docker-compose exec laravel_app php artisan key:generate
    ```
  </Step>

  <Step title="Configure environment variables">
    Edit the `.env` file with your settings (see sections below).
  </Step>
</Steps>

### Required Environment Variables

#### Application Settings

```env .env
# Application name displayed in UI
APP_NAME="7POINTS.NET"

# Application environment: local, staging, production
APP_ENV=local

# Application key (generated by php artisan key:generate)
APP_KEY=base64:YOUR_GENERATED_KEY_HERE

# Enable debug mode (set to false in production)
APP_DEBUG=true

# Application URL
APP_URL=http://localhost:8080
```

<Warning>
  Always set `APP_DEBUG=false` in production to prevent sensitive information leakage.
</Warning>

#### Database Connection

```env .env
# Database driver (pgsql for PostgreSQL)
DB_CONNECTION=pgsql

# Database host (use 'postgres' in Docker, 'localhost' outside)
DB_HOST=postgres

# Database port
DB_PORT=5432

# Database name
DB_DATABASE=postgres

# Database credentials
DB_USERNAME=odoo
DB_PASSWORD=myodoo
```

<Note>
  When running Laravel outside Docker, change `DB_HOST=postgres` to `DB_HOST=localhost`.
</Note>

#### Session and Cache Configuration

```env .env
# Session driver: file, cookie, database, redis
SESSION_DRIVER=file
SESSION_LIFETIME=120

# Cache driver: file, database, redis
CACHE_DRIVER=file

# Queue driver: sync, database, redis, sqs
QUEUE_CONNECTION=sync
```

#### Odoo Integration Settings

```env .env
# Odoo connection settings
ODOO_URL=http://odoo:8069
ODOO_DATABASE=postgres
ODOO_USERNAME=admin
ODOO_PASSWORD=admin
```

#### Mail Configuration (Optional)

```env .env
# Mail driver: smtp, sendmail, mailgun, ses, postmark
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=2525
MAIL_USERNAME=null
MAIL_PASSWORD=null
MAIL_ENCRYPTION=null
MAIL_FROM_ADDRESS="hello@7points.net"
MAIL_FROM_NAME="${APP_NAME}"
```

#### Logging Configuration

```env .env
# Log channel: stack, single, daily, slack, syslog, errorlog
LOG_CHANNEL=stack
LOG_LEVEL=debug
```

## Odoo Environment Configuration

Odoo environment variables are configured in `docker-compose.yaml`:

```yaml docker-compose.yaml
odoo:
  image: odoo:17.0
  environment:
    # PostgreSQL host (container name)
    - HOST=postgres
    
    # Database username
    - USER=odoo
    
    # Database password
    - PASSWORD=myodoo
    
    # Optional: Specific database to use
    # - POSTGRES_DB=odoo_production
```

### Odoo Configuration File (Optional)

For advanced configuration, create `odoo/config/odoo.conf`:

```ini odoo.conf
[options]
admin_passwd = YOUR_STRONG_MASTER_PASSWORD
db_host = postgres
db_port = 5432
db_user = odoo
db_password = myodoo
addons_path = /mnt/extra-addons
data_dir = /var/lib/odoo

# Workers (set to 0 for development)
workers = 4
max_cron_threads = 2

# Limits
limit_memory_hard = 2684354560
limit_memory_soft = 2147483648
limit_request = 8192
limit_time_cpu = 600
limit_time_real = 1200

# Logging
log_level = info
logfile = /var/log/odoo/odoo.log
```

Mount the configuration file in docker-compose:

```yaml
odoo:
  volumes:
    - ./odoo/config/odoo.conf:/etc/odoo/odoo.conf
  command: -- -c /etc/odoo/odoo.conf
```

## PostgreSQL Environment Configuration

PostgreSQL is configured via environment variables in `docker-compose.yaml`:

```yaml docker-compose.yaml
postgres:
  image: postgres:15
  environment:
    # Default database name
    - POSTGRES_DB=postgres
    
    # PostgreSQL superuser name
    - POSTGRES_USER=odoo
    
    # PostgreSQL superuser password
    - POSTGRES_PASSWORD=myodoo
    
    # Optional: Initialization scripts
    # - POSTGRES_INITDB_ARGS=--encoding=UTF-8
```

## Generating APP_KEY

The Laravel `APP_KEY` is used for encryption and must be set before deploying.

<CodeGroup>
```bash Local Development
cd laravel
php artisan key:generate
```

```bash Docker
docker-compose exec laravel_app php artisan key:generate
```

```bash Manual Generation
# Generate a random base64 key
php -r "echo 'base64:' . base64_encode(random_bytes(32)) . PHP_EOL;"
```
</CodeGroup>

## Environment-Specific Configurations

### Development Environment

```env .env.development
APP_ENV=local
APP_DEBUG=true
LOG_LEVEL=debug

DB_HOST=postgres
ODOO_URL=http://odoo:8069
```

### Production Environment

```env .env.production
APP_ENV=production
APP_DEBUG=false
LOG_LEVEL=error

DB_HOST=your-production-db-host
ODOO_URL=https://odoo.yourdomain.com

# Use strong passwords
DB_PASSWORD=STRONG_RANDOM_PASSWORD
ODOO_PASSWORD=STRONG_RANDOM_PASSWORD
```

## Security Best Practices

<Warning>
  Never commit `.env` files to version control. Add `.env` to `.gitignore`.
</Warning>

### Password Requirements

1. **Minimum 16 characters** for production passwords
2. **Use random generation**: `openssl rand -base64 32`
3. **Different passwords** for each service
4. **Rotate regularly** (every 90 days)

### Securing Sensitive Variables

<Steps>
  <Step title="Use environment management tools">
    Consider using:
    - AWS Secrets Manager
    - HashiCorp Vault
    - Laravel Envoyer
  </Step>

  <Step title="Encrypt .env file in CI/CD">
    ```bash
    # Encrypt for deployment
    openssl enc -aes-256-cbc -in .env -out .env.enc
    
    # Decrypt on server
    openssl enc -d -aes-256-cbc -in .env.enc -out .env
    ```
  </Step>

  <Step title="Restrict file permissions">
    ```bash
    chmod 600 .env
    chown www-data:www-data .env
    ```
  </Step>
</Steps>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="APP_KEY not set">
    **Error**: `No application encryption key has been specified.`
    
    **Solution**:
    ```bash
    php artisan key:generate
    ```
  </Accordion>

  <Accordion title="Database connection refused">
    **Error**: `SQLSTATE[08006] Connection refused`
    
    **Solution**:
    - Check `DB_HOST` is correct (`postgres` in Docker)
    - Ensure PostgreSQL container is running
    - Verify database credentials
  </Accordion>

  <Accordion title="Odoo connection failed">
    **Error**: Cannot connect to Odoo
    
    **Solution**:
    - Verify `ODOO_URL=http://odoo:8069` (not localhost in Docker)
    - Ensure Odoo container is running
    - Check Odoo credentials
  </Accordion>
</AccordionGroup>
