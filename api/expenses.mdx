## Overview

The Expenses API allows apartment members to track shared expenses, manage payments, and settle debts.

<Note>
All expense endpoints require authentication and an active apartment selected in session.
</Note>

## List Expenses

Get all expenses for the current apartment with optional date filtering.

```
GET /gastuak
```

### Query Parameters

<ParamField query="fecha" type="string">
  Filter by date (format: `YYYY-MM` or `YYYY-MM-DD`)
</ParamField>

### Request Example

```bash
curl -X GET "https://your-domain.com/gastuak?fecha=2026-03" \
  -H "Accept: application/json" \
  --cookie cookies.txt
```

### Response Example

```json
{

    "id": 1,
    "izena": "Downtown Apartment",
    "kodigoa": "04823",

      }
    ],
    "gastos": [
      {
        "IdGasto": 1,
        "IdUsuario": 5,
```

## Add Expense

Create a new expense in the current apartment.

```
POST /gastuak/addGastua
```

### Request Parameters

<ParamField path="Nombre" type="string" required>
  Expense name/description (max 255 characters)
</ParamField>

<ParamField path="Cantidad" type="number" required>
  Expense amount (must be >= 0)
</ParamField>

<ParamField path="Fecha" type="date" required>
  Expense date (format: YYYY-MM-DD)
</ParamField>

<ParamField path="IdUsuario" type="integer" required>
  ID of user who paid for the expense
</ParamField>

### Request Example

```bash
curl -X POST https://your-domain.com/gastuak/addGastua \
  -H "Content-Type: application/json" \
  -H "X-CSRF-TOKEN: {csrf-token}" \
  --cookie cookies.txt \
  -d '{
    "Nombre": "Electricity Bill",
    "Cantidad": 85.50,
    "Fecha": "2026-03-05",
    "IdUsuario": 5
  }'
```

### Response

Redirects to `/gastuak` (expense list).

<ResponseField name="IdGasto" type="integer">
  Auto-generated expense ID
</ResponseField>

<ResponseField name="IdPiso" type="integer">
  Automatically set from session
</ResponseField>

<ResponseField name="odoo_id" type="integer" nullable>
  Odoo ERP integration ID (synced asynchronously)
</ResponseField>

## Get Expenses by Apartment and User

Retrieve expenses filtered by apartment and user.

```
GET /gastos/{idPisua}/{idErabiltzailea}
```

### Path Parameters

<ParamField path="idPisua" type="integer" required>
  Apartment ID
</ParamField>

<ParamField path="idErabiltzailea" type="integer" required>
  User ID
</ParamField>

### Request Example

```bash
curl -X GET https://your-domain.com/gastos/1/5 \
  -H "Accept: application/json" \
  --cookie cookies.txt
```

### Response

Same structure as List Expenses, filtered by user.

## Update Expense

Update an existing expense.

```
PUT /gastuak/editar/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Expense ID
</ParamField>

### Request Parameters

<ParamField path="Nombre" type="string" required>
  Updated expense name (max 255 characters)
</ParamField>

<ParamField path="Cantidad" type="number" required>
  Updated amount (must be >= 0)
</ParamField>

<ParamField path="Fecha" type="date" required>
  Updated date (format: YYYY-MM-DD)
</ParamField>

<ParamField path="IdUsuario" type="integer" required>
  Updated user ID (must exist in users table)
</ParamField>

### Request Example

```bash
curl -X PUT https://your-domain.com/gastuak/editar/1 \
  -H "Content-Type: application/json" \
  -H "X-CSRF-TOKEN: {csrf-token}" \
  --cookie cookies.txt \
  -d '{
    "Nombre": "Updated Groceries",
    "Cantidad": 50.00,
    "Fecha": "2026-03-02",
    "IdUsuario": 5
  }'
```

### Response

Redirects to `/gastuak` (expense list).

## Delete Expense

Delete an expense permanently.

```
DELETE /gastos/deleteGasto/{id}
```

### Path Parameters

<ParamField path="id" type="integer" required>
  Expense ID to delete
</ParamField>

### Request Example

```bash
curl -X DELETE https://your-domain.com/gastos/deleteGasto/1 \
  -H "X-CSRF-TOKEN: {csrf-token}" \
  --cookie cookies.txt
```

### Response

Redirects to `/gastuak` (expense list).

<Info>
If the expense was synced to Odoo, deletion will be propagated asynchronously.
</Info>

## Settle Debt

Record a payment to settle debt between apartment members.

```
POST /pagos/saldar
```

### Request Parameters

<ParamField path="deudor_id" type="integer" required>
  ID of user who owes money (must exist in users table)
</ParamField>

<ParamField path="acreedor_id" type="integer" required>
  ID of user who is owed money (must exist in users table)
</ParamField>

<ParamField path="cantidad" type="number" required>
  Payment amount (must be >= 0.01)
</ParamField>

### Request Example

```bash
curl -X POST https://your-domain.com/pagos/saldar \
  -H "Content-Type: application/json" \
  -H "X-CSRF-TOKEN: {csrf-token}" \
  --cookie cookies.txt \
  -d '{
    "deudor_id": 7,
    "acreedor_id": 5,
    "cantidad": 25.00
  }'
```

### Response

Redirects back to previous page.

<ResponseField name="piso_id" type="integer">
  Automatically set from session
</ResponseField>

<ResponseField name="deudor_id" type="integer">
  Debtor user ID
</ResponseField>

<ResponseField name="acreedor_id" type="integer">
  Creditor user ID
</ResponseField>

<ResponseField name="cantidad" type="decimal">
  Payment amount
</ResponseField>
