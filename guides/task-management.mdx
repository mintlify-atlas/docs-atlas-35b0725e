This guide covers the complete task management system (Zereginak) in 7POINTS.NET, helping you organize household chores and responsibilities among roommates.

## Overview

The task management system allows you to:
- Create and assign household tasks to specific members
- Track task status (to-do, in-progress, completed)
- Set start dates for tasks
- View all tasks in a centralized list
- Edit or delete tasks as needed
- Keep everyone accountable for their responsibilities

<Info>
All tasks are apartment-specific. You'll only see tasks for your currently active apartment.
</Info>

## Accessing Task Management

Navigate to **Zereginak** from the main navigation menu, or go directly to `/zereginak`.

You must have an active apartment selected (stored in your session) to view or create tasks.

## Creating a New Task

<Steps>
  <Step title="Navigate to Task Creation">
    From the main Zereginak page:
    1. Click the **"Sortu"** or **"New Task"** button
    2. Or navigate directly to `/zereginak/sortu`
    
    You'll see a clean form titled **"Zeregina sortu"** (Create Task).
  </Step>

  <Step title="Fill in Task Details">
    Complete the following fields:
    
    **Zereginaren izena (Task Name)**
    - Brief description of the task
    - Maximum 50 characters
    - Character counter displays remaining space
    - Examples: "Komuna garbitu" (Clean bathroom), "Take out trash", "Buy groceries"
    
    **Deskripzioa (Description)**
    - Optional detailed notes about the task
    - Maximum 255 characters
    - Can include specific instructions or requirements
    - Displayed in a multi-line textarea
    
    **Nork egingo du? (Who will do it?)**
    - Select which apartment member is assigned the task
    - Dropdown shows all current roommates
    - Required field - every task must have an assignee
    
    **Hasiera-data (Start Date)**
    - When the task should begin or be completed
    - Date picker in YYYY-MM-DD format
    - Can be present or future dates
    - Required field
    
    <Tip>
    Write clear task names that anyone can understand at a glance. Include specifics in the description field if needed.
    </Tip>
  </Step>

  <Step title="Submit the Task">
    Click **"Zeregina gorde"** (Save Task) to create the task.
    
    The system will:
    - Validate all required fields
    - Create the task in the `zereginak` table
    - Link it to your current apartment
    - Associate the assigned member via the pivot table
    - Set initial status to `egiteko` (to-do)
    - Queue synchronization with Odoo ERP
    - Redirect to the task list (`/zereginak`)
    
    If you change your mind, click **"Ezeztatu eta itzuli"** (Cancel and return).
  </Step>
</Steps>

## Task Statuses

Every task has one of three statuses:

<CodeGroup>
```text To-Do
egiteko - Task has been created but not started
```

```text In Progress
egiten - Task is currently being worked on
```

```text Completed
eginda - Task has been finished
```
</CodeGroup>

<Note>
New tasks automatically start with `egiteko` status. Members can update the status as they work on tasks.
</Note>

## Viewing Tasks

### Task List View

The main Zereginak page (`/zereginak`) displays all tasks for your active apartment:

**Information shown for each task:**
- Task name (izena)
- Description (deskripzioa)
- Assigned member (erabiltzaileak)
- Start date (hasiera_data)
- Current status (egoera)
- Actions (edit, delete)

**Features:**
- Tasks are loaded with their assigned users via the `erabiltzaileak` relationship
- Filtered automatically by your apartment (`pisua_id` from session)
- Displays apartment name in the page title

### Task Organization

Tasks may be displayed:
- In a list or table format
- Grouped by status (to-do, in-progress, completed)
- With color coding by status or assignee
- With quick action buttons

<Tip>
Check the task list regularly to see what needs to be done and what your assigned tasks are.
</Tip>

## Marking Tasks as In-Progress

To indicate you've started working on a task:

<Steps>
  <Step title="Find Your Task">
    Locate the task assigned to you in the task list.
  </Step>

  <Step title="Edit the Task">
    Click the edit button/icon next to the task, or click on the task itself.
  </Step>

  <Step title="Change Status">
    Update the **Egoera** (Status) field to **egiten** (in-progress).
  </Step>

  <Step title="Save">
    Submit the form to update the task status.
  </Step>
</Steps>

This helps roommates see what's actively being worked on.

## Marking Tasks as Completed

When you finish a task:

<Steps>
  <Step title="Access Task Edit">
    Click the edit button for the completed task.
  </Step>

  <Step title="Set Status to Completed">
    Change the **Egoera** field to **eginda** (completed).
  </Step>

  <Step title="Save Changes">
    Submit the form to mark the task as done.
  </Step>
</Steps>

<Info>
Completed tasks typically remain visible in the list so everyone can see what's been accomplished. They may be styled differently (e.g., grayed out, strikethrough) to distinguish them from active tasks.
</Info>

## Editing Tasks

<Steps>
  <Step title="Navigate to Edit Form">
    From the task list, click the edit icon/button next to the task you want to modify.
    
    You'll be taken to `/zereginak/{zeregina}/editatu`.
  </Step>

  <Step title="Modify Task Details">
    You can change:
    - **Izena** (Task name)
    - **Deskripzioa** (Description)
    - **Egoera** (Status: egiteko, egiten, eginda)
    - **Hasiera_data** (Start date)
    - **Erabiltzailea_id** (Assigned member)
    
    All original validation rules apply.
  </Step>

  <Step title="Save Changes">
    Click the save button to update the task.
    
    The system will:
    - Validate changes
    - Update the task in the database
    - Update the user assignment in the pivot table (using `sync`)
    - Re-sync with Odoo via `SyncZereginakToOdoo`
    - Redirect back to `/zereginak`
  </Step>
</Steps>

<Warning>
The edit form verifies that the task belongs to your current apartment (`pisua_id` check). You cannot edit tasks from other apartments, even if you know the URL.
</Warning>

## Deleting Tasks

<Steps>
  <Step title="Locate the Task">
    Find the task you want to delete in the task list.
  </Step>

  <Step title="Click Delete">
    Click the trash/delete icon next to the task.
    
    A confirmation dialog appears with the message:
    **"Ziur zaude zeregin hau ezabatu nahi duzula?"** (Are you sure you want to delete this task?)
  </Step>

  <Step title="Confirm Deletion">
    Click OK/Confirm to permanently delete the task.
    
    The system will:
    - Verify the task belongs to your apartment (security check)
    - Delete the task from the database
    - Queue deletion in Odoo (if synced)
    - Redirect back to the task list
  </Step>
</Steps>

<Warning>
Deleting a task is permanent and cannot be undone. The task and all its history will be removed. Only delete tasks that were created in error or are no longer needed.
</Warning>

## Assigning Tasks

### Initial Assignment

When creating a task, you select the assigned member from a dropdown of all apartment members (kideak).

**Best practices:**
- Assign tasks fairly across all roommates
- Consider each person's schedule and availability
- Rotate recurring tasks among members
- Discuss assignments in person when possible

### Reassigning Tasks

To change who is assigned to a task:

1. Edit the task
2. Change the **Nork egingo du?** (Who will do it?) dropdown
3. Save the changes

The new assignee will immediately see the task on their list.

<Tip>
Use the description field to note if a task is being reassigned and why. This provides context and prevents confusion.
</Tip>

## Recurring Tasks

<Note>
The current system does NOT have built-in support for automatically recurring tasks.
</Note>

To handle recurring tasks:

**Option 1: Manual Recreation**
- Create a new task each week/month
- Copy the details from the previous occurrence
- Assign to the next person in rotation

**Option 2: Single Task with Updates**
- Create one task (e.g., "Weekly Bathroom Cleaning")
- Update the status and assignee each week
- Use the description to note rotation schedule

**Option 3: Create Multiple in Advance**
- Create several tasks at once with future dates
- Assign them in rotation
- They'll appear as the dates approach

<Info>
Future versions may include recurring task templates and automatic rotation features.
</Info>

## Calendar View

<Warning>
The current implementation does NOT include a calendar view UI component.
</Warning>

Tasks have start dates (`hasiera_data`), which could be visualized in a calendar format, but this is not currently built into the interface.

If you need calendar functionality:
- Tasks can be filtered or sorted by date in the task list
- You could export task data and import into an external calendar
- Future updates may add calendar visualization

## Common Questions

<AccordionGroup>
  <Accordion title="Can I assign a task to multiple people?">
    No, the current system assigns each task to exactly one person (`erabiltzailea_id`).
    
    For tasks requiring multiple people:
    - Create separate tasks for each person
    - Mention the collaboration in the description
    - Or assign to one person who coordinates with others
  </Accordion>
  
  <Accordion title="What happens to tasks if someone leaves the apartment?">
    Tasks assigned to a removed member:
    - Remain in the database
    - May display the person's ID or a placeholder
    - Should be reassigned to current members
    - Or deleted if no longer relevant
    
    Consider reviewing and cleaning up tasks when membership changes.
  </Accordion>
  
  <Accordion title="Can I see tasks from other apartments?">
    No. Tasks are strictly filtered by your active apartment:
    - Only tasks matching `session('pisua_id')` are shown
    - Editing requires the task's `pisua_id` to match your session
    - This prevents cross-apartment access
    
    If you're in multiple apartments, switch apartments to see different task lists.
  </Accordion>
  
  <Accordion title="How do I track task history?">
    The current system does not maintain a detailed history log.
    
    Task changes (status updates, reassignments) overwrite the existing record.
    
    For audit trails:
    - Check the Odoo backend if synchronized
    - Use the `updated_at` timestamp to see when last modified
    - Consider keeping manual logs for important tasks
  </Accordion>
</AccordionGroup>

## Technical Details

### Key Routes
- **View tasks**: `GET /zereginak` → `ZereginakController@index`
- **Create form**: `GET /zereginak/sortu` → `ZereginakController@create`
- **Store task**: `POST /zereginak` → `ZereginakController@store`
- **Edit form**: `GET /zereginak/{zeregina}/editatu` → `ZereginakController@edit`
- **Update task**: `PUT /zereginak/{zeregina}` → `ZereginakController@update`
- **Delete task**: `DELETE /zereginak/{zeregina}` → `ZereginakController@destroy`

### Data Model
- **Zereginak** table: Stores tasks with fields `izena`, `deskripzioa`, `pisua_id`, `egoera`
- **Pivot table** (`erabiltzaileak` relationship): Links tasks to users with `hasiera_data`
- **Status enum**: `egiteko`, `egiten`, `eginda` (enforced in validation)

### Controller Location
`app/Http/Controllers/ZereginakController.php` (lines 1-176)

### Security
- All routes require `auth` and `verified` middleware
- Tasks are automatically scoped to the active apartment
- Edit/delete operations verify `pisua_id` matches session to prevent unauthorized access
